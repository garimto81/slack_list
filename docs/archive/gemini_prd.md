PRD: GitHub-Slack 업무 현황 자동화 및 번역 시스템1. 개요 (Overview)프로젝트명: Project Translator (가칭)작성일: 2024.XX.XX담당자: PM/PO 이름배경: 개발팀은 GitHub에서 커밋 로그로 소통하고, 비즈니스팀은 슬랙에서 업무를 확인한다. 이 과정에서 용어의 차이로 인해 불필요한 커뮤니케이션 비용("이게 무슨 뜻인가요?", "어디까지 진행됐나요?")이 발생하며, 슬랙 채널이 알림 메시지로 도배되는 문제가 있다.목표:GitHub의 작업 완료(Merge) 이벤트를 감지하여 슬랙 리스트 상태를 실시간 동기화한다.개발 용어(Commit Message)를 **일반인이 이해하기 쉬운 요약문으로 자동 변환(AI)**한다.채팅 메시지가 아닌 Dashboard(Slack List) 형태로 최신 상태를 유지한다.2. 타겟 유저 (Target Audience)개발자: 별도의 보고서 작성 없이 코드를 푸시/머지하는 것만으로 현황 공유를 끝내고 싶음.기획/마케팅/경영진: 복잡한 기술 용어 대신 "로그인 오류 해결됨", "결제 페이지 오픈" 등 직관적인 현황을 보고 싶음.3. 핵심 기능 요구사항 (Functional Requirements)3.1. 트리거 및 매칭 (Trigger & Matching)[GitHub] Pull Request Merged 또는 Issue Closed 이벤트 발생 시 워크플로우가 시작되어야 한다.[식별자] GitHub PR 제목이나 브랜치명에 포함된 **티켓 번호(예: PROJ-102)**를 파싱하여, 슬랙 리스트 내의 해당 티켓과 매칭해야 한다.예외 처리: 티켓 번호가 없으면 '기타 유지보수' 항목으로 분류하거나 업데이트를 스킵한다.3.2. AI 번역 및 요약 (Translation Engine)GitHub의 PR Title, PR Body, Commit Messages를 수집하여 LLM(ChatGPT/Claude API)에 전송한다.프롬프트 요구사항:기술적 세부 사항(Refactoring, API call 등) 제외.사용자 관점의 가치(Value) 위주로 1문장 요약.톤앤매너: 비즈니스 친화적, 완료형 문장.예시: "fix: auth token expiration logic" -> "자동 로그인 풀림 현상 수정 완료"3.3. 슬랙 리스트 업데이트 (Slack List Action)식별된 슬랙 리스트의 Row(행)를 찾아 다음 필드를 업데이트한다.상태(Status): 진행 중 -> 완료 (또는 QA 대기)업무 내용(Description): AI가 요약한 텍스트로 덮어쓰기(또는 비고란 추가).마지막 업데이트(Date): 현재 시간.채널에 별도의 메시지 알림을 보내지 않는다 (Quiet Update).4. 유저 스토리 (User Stories)ID주체(Actor)상황(Situation)요구사항(Need)목적(Benefit)US-1개발자기능 개발을 완료하고 코드를 머지했을 때슬랙에 별도로 들어가 "완료했습니다"라고 쓰지 않아도 되기를 원함업무 집중도 향상US-2기획자현재 프로젝트 진행 상황이 궁금할 때개발자에게 물어보지 않고 슬랙 리스트만 봐도 알기를 원함커뮤니케이션 비용 절감US-3마케팅팀새로운 기능이 배포되었을 때"API 수정"이 아니라 "회원가입 기능 개선"이라고 적혀있기를 원함즉시 업무 파악 가능5. 데이터 흐름도 (Data Flow)[GitHub] 개발자가 PR Merge (Title: feat: add social login, Branch: PROJ-101)[Middleware] (Zapier/Make) Webhook 수신 -> 티켓 ID PROJ-101 추출[AI Engine] 입력: "feat: add social login" -> 출력: "카카오/구글 소셜 로그인 기능 추가 완료"[Slack API] 슬랙 리스트에서 PROJ-101 ID를 가진 아이템 검색[Slack List] 해당 아이템의 상태를 Done, 설명을 카카오/구글 소셜 로그인 기능 추가 완료로 변경6. 상세 스펙 및 제약사항 (Constraints)Slack Plan: Slack Lists 기능을 사용하기 위해 Pro 플랜 이상이어야 함.비용: OpenAI API (또는 Zapier AI 등) 사용에 따른 소액의 비용 발생 가능.Latency: GitHub 액션 후 슬랙 반영까지 1분 이내여야 함.오류 처리: AI가 요약에 실패하거나 매칭되는 리스트가 없을 경우, 지정된 관리자에게 DM으로 알림을 보낸다.7. 화면 예시 (UI Mockup)[변경 전: Slack List]| ID | 업무명 | 상태 | 비고 ||:---:|:---|:---:|:---|| PROJ-101 | 소셜 로그인 개발 | 진행 중 | - |[변경 후: Slack List] (자동 업데이트 됨)| ID | 업무명 | 상태 | 비고 ||:---:|:---|:---:|:---|| PROJ-101 | 소셜 로그인 개발 | 완료 | 카카오/구글 로그인 기능 추가됨 (by AI) |
